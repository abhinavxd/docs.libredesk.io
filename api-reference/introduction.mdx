## Authentication Methods

LibreDesk supports multiple authentication methods to fit your integration needs:

<AccordionGroup>
  <Accordion title="API Key Authentication" icon="key">
    Server-to-server authentication using API keys. Generate and manage API keys through the agent endpoints.
    
    ```json
    {
      "headers": {
        "Authorization": "Bearer your-api-key"
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Basic Authentication" icon="lock">
    Simple username and password authentication for direct API access.
    
    ```json
    {
      "headers": {
        "Authorization": "Basic base64(username:password)"
      }
    }
    ```
  </Accordion>
  
  <Accordion title="OAuth/OIDC" icon="shield">
    Enterprise-grade authentication with OpenID Connect support for single sign-on integration.
    
    ```json
    {
      "security": [
        {
          "oidc": ["openid", "profile", "email"]
        }
      ]
    }
    ```
  </Accordion>
</AccordionGroup>

## Response Format

All API responses follow a consistent JSON structure with standard HTTP status codes:

```json
{
  "data": {},
  "message": "Success",
  "status": 200
}
```

## Error Handling

LibreDesk uses conventional HTTP response codes to indicate the success or failure of requests:

- `200` - Success
- `400` - Bad Request - Invalid parameters or malformed request
- `401` - Unauthorized - Authentication required or invalid
- `403` - Forbidden - Insufficient permissions
- `404` - Not Found - Resource does not exist
- `500` - Internal Server Error - Server-side error
